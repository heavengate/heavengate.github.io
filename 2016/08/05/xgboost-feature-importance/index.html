XGBoost支持的特征重要性算法

发表于 2016-08-05   |   分类于 ML   |  

1. 概述

　　XGBoost是最近很牛逼的boost实现，在小规模数据和特征的分类模型中有不俗的表现效果，在很多数据挖掘竞赛中也有一席之地。基于C++底层实现的XGBoost有较快的运行速度，同时XGBoost支持常用的数据处理语言的接口，诸如python、Java，R，Scala等，是目前机器学习分类在工业界运用得较多的一种包。
　　今天主要通过python的源码解析一下XGBoost的特征重要性算法。

2. 重要性算法

　　本位的源码解析基于GitHub上的XGBoost源码的python包，链接：“XGBoost-python”。源码的xgboost/core.py中get_score()函数头部注释如下。
```python
def get_score(self, fmap=’’, importance_type=’weight’):
“””Get feature importance of each feature.
Importance type can be defined as:
‘weight’ - the number of times a feature is used to split the data across all trees.
‘gain’ - the average gain of the feature when it is used in trees
‘cover’ - the average coverage of the feature when it is used in trees

Parameters
----------
fmap: str (optional)
   The name of feature map file
"""

　　从上述代码注释可以看出，XGBoost支持三种特征重要性计算算法如下：

  • weight: 特征在所有树中出现的总次数
  • gain: 特征在所有树中出现对于模型的平均提升（Gini不纯度增益）
  • cover: 特征在所有树中平均覆盖的样本数

4. 特征重要性源码分析

　　XGBoost的python包所有涉及到特征重要性的函数最后均是调用get_score()函数，下面逐步分析其源码。
　　首先，get_score()判断importance_type，若其不为’weight’、’gain’、’cover’之一则报错。
```python
def get_score(self, fmap=’’, importance_type=’weight’):
if importance_type not in [‘weight’, ‘gain’, ‘cover’]:
msg = “importance_type mismatch, got ‘{}’, expected ‘weight’, ‘gain’, or ‘cover’”
raise ValueError(msg.format(importance_type))

　　当importance_type为weight时，调用get_dump()函数读入之前dump的model。

# if it's weight, then omap stores the number of missing values
if importance_type == 'weight':
    # do a simpler tree dump to save time
    trees = self.get_dump(fmap, with_stats=False)

　　对于tree里的每个tree，即为model里的每个Booster，读取每个节点行，匹配行里的[]之间的字符，提取出来之后，判断该特征是否在行中，若特征出现在此行，即为该节点选用此特征，特征计数加一，最后返回计数的fmap。　　
fmap = {}
for tree in trees:
for line in tree.split(‘\n’):

        # look for the opening square bracket
        arr = line.split('[')
        # if no opening bracket (leaf node), ignore this line
        if len(arr) == 1:
            continue
        # extract feature name from string between []
        fid = arr[1].split(']')[0].split('<')[0]
        if fid not in fmap:
            # if the feature hasn't been seen yet
            fmap[fid] = 1
        else:
            fmap[fid] += 1
return fmap

　　当特征不是weight时，此时模型在训练时会将对应的gain值或cover值付于节点行后，则只需匹配出该值做相应的运算即可，原理同上。
else:
trees = self.get_dump(fmap, with_stats=True)
importance_type += ‘=’
fmap = {}
gmap = {}
for tree in trees:
for line in tree.split(‘\n’):

        # look for the opening square bracket
        arr = line.split('[')
        # if no opening bracket (leaf node), ignore this line
        if len(arr) == 1:
            continue
        # look for the closing bracket, extract only info within that bracket
        fid = arr[1].split(']')
        # extract gain or cover from string after closing bracket
        g = float(fid[1].split(importance_type)[1].split(',')[0])
        # extract feature name from string before closing bracket
        fid = fid[0].split('<')[0]
        if fid not in fmap:
            # if the feature hasn't been seen yet
            fmap[fid] = 1
            gmap[fid] = g
        else:
            fmap[fid] += 1
            gmap[fid] += g
# calculate average value (gain/cover) for each feature
for fid in gmap:
    gmap[fid] = gmap[fid] / fmap[fid]
return gmap
