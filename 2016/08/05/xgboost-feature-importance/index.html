XGBoost支持的特征重要性算法

发表于 2016-08-05   |   分类于 ML   |  

1. 概述

　　XGBoost是最近很牛逼的boost实现，在小规模数据和特征的分类模型中有不俗的表现效果，在很多数据挖掘竞赛中也有一席之地。基于C++底层实现的XGBoost有较快的运行速度，同时XGBoost支持常用的数据处理语言的接口，诸如python、Java，R，Scala等，是目前机器学习分类在工业界运用得较多的一种包。
　　今天主要通过python的源码解析一下XGBoost的特征重要性算法。

2. 重要性算法

　　本位的源码解析基于GitHub上的XGBoost源码的python包，链接：“XGBoost-python”。源码的xgboost/core.py中get_score()函数头部注释如下。

　　从上述代码注释可以看出，XGBoost支持三种特征重要性计算算法如下：

  • weight: 特征在所有树中出现的总次数
  • gain: 特征在所有树中出现对于模型的平均提升（Gini不纯度增益）
  • cover: 特征在所有树中平均覆盖的样本数

3. Model保存格式

　　在XGBoost中计算特征重要性时，get_score()会调用get_dump()函数将之前dump的model读进来，根据dump的数据顺序计算特征的重要性，因此我们需要知道XGBoost中dump_model()的存储格式，才能进一步分析计算特征重要性的算法。

　　在节点行，首先声明节点序号，中括号里写明该节点采用第几个特征（如f34即为训练数据的第34个特征），同时表明特征值划分条件，中括号后分别标明划分区间对用的子节点。

4. 特征重要性源码分析

　　XGBoost的python包所有涉及到特征重要性的函数最后均是调用get_score()函数，下面逐步分析其源码。
　　首先，get_score()判断importance_type，若其不为’weight’、’gain’、’cover’之一则报错。

　　当importance_type为weight时，调用get_dump()函数读入之前dump的model。

　　对于tree里的每个tree，即为model里的每个Booster，读取每个节点行，匹配行里的[]之间的字符，提取出来之后，判断该特征是否在行中，若特征出现在此行，即为该节点选用此特征，特征计数加一，最后返回计数的fmap。　　

　　当特征不是weight时，此时模型在训练时会将对应的gain值或cover值付于节点行后，则只需匹配出该值做相应的运算即可，原理同上。

5. 总结

　　以上即为python包中计算特征重要性的三种算法的原理和实现，其中gain值和cover值是模型在训练过程中已实现的，计算过程中只需读取该值即可，若要知道该值的实际计算过程，还需要深入XGBoost底层的C++代码去阅读，有机会再和大家分享。
